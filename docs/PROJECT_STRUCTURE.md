# Структура проекта

Проект организован как пакет `app`. Имена файлов сохранены без переименования: файлы только разложены по папкам по зоне ответственности.

## Основные папки

- `app/` — исходный код приложения.
- `app/gui/` — PyQt-интерфейс: главное окно, калькулятор, визуальный редактор, диалоги, стили и экранные утилиты.
- `app/core/` — расчетное ядро, модели данных, валидаторы, выбор методики, нормативные методы и анализ вкладов.
- `app/formulas/` — генерация, разбор, упаковка и рендеринг формул.
- `app/reports/` — экспорт отчетов в HTML, TXT, PDF, DOCX и XLSX.
- `app/import_export/` — загрузка/сохранение схем, импорт внешних JSON/YAML, экспорт изображений схем и безопасная работа с файлами.
- `app/demo/` — демо-сценарии, встроенные шаблоны и `db_modules.json`.
- `app/utils/` — общие константы, логирование и helpers путей.
- `examples/` — пользовательские и демонстрационные входные JSON/YAML.
- `docs/` — документация проекта.
- `resources/` — иконки и статические ресурсы приложения. Папка сохранена без переименования для совместимости.
- `tests/` — автоматические тесты.
- `tools/` — вспомогательные скрипты.
- `backup/` и `deliverables/` — резервные копии и поставочные материалы.

## Работа с путями

Для ресурсов проекта используйте `app/utils/paths.py`:

- `project_root()` — корень проекта или папка приложения в frozen-сборке.
- `app_root()` — папка `app`.
- `resource_path(*parts)` — файлы из `resources/`.
- `examples_path(*parts)` — файлы из `examples/`.
- `docs_path(*parts)` — файлы из `docs/`.
- `deliverables_path(*parts)` — файлы из `deliverables/`.
- `demo_path(*parts)` — файлы из `app/demo/`.

Не привязывайте код к текущей рабочей директории через `Path.cwd()` для проектных ресурсов.

## Запуск

Основной запуск:

```bash
python -m app.main
```

Совместимый запуск из корня:

```bash
python main.py
```

Старый запуск `python gui_main.py` сохранен через маленький wrapper.

На Windows также доступны:

```powershell
.\run_app.ps1
```

```bat
run_app.bat
```

## Тесты

```bash
pytest
```

Если среда ограничивает системный `%TEMP%`, можно задать отдельный `--basetemp` в доступной папке.

## Сборка

PyInstaller использует `ReliabilityApp.spec`. Точка входа сборки — `app/main.py`; ресурсы берутся из `resources/`, примеры из `examples/`, демо-данные из `app/demo/`.

## Куда добавлять новый код

- Новый GUI-модуль — в `app/gui/`.
- Новую расчетную функцию или модель — в `app/core/`.
- Новую формулу, рендерер или генератор формул — в `app/formulas/`.
- Новый экспорт отчета — в `app/reports/`.
- Новый импорт/экспорт файлов — в `app/import_export/`.
- Новый демо-сценарий или шаблон — в `app/demo/`.
- Общий helper без доменной логики — в `app/utils/`.
