# Тестирование и проверка

## 1. Запуск автотестов

```bash
pytest
```

Если команда `pytest` не найдена, сначала установите зависимости:

```bash
pip install -r requirements.txt
```

## 2. Набор тестов

### `tests/test_backend.py`

Проверяет расчетное ядро:

- F1.1 на ручной формуле последовательного соединения;
- F2.2 для разных сценариев резерва с восстановлением;
- F5.1 на неотрицательные показатели;
- F6.3 с документированным ограничением по `T0`.

### `tests/test_formula_builder.py`

Проверяет генератор формул:

- пустая схема;
- один блок;
- два последовательных блока;
- два параллельных блока;
- последовательность перед параллельным участком;
- параллель подсхем;
- вложенная схема;
- резервирование;
- длинная последовательная цепочка;
- широкая параллельная схема;
- большая комбинированная схема;
- неиспользованный блок.

Ключевой инвариант: AST, формула, обозначения, вычислительное представление и объяснение должны содержать один и тот же набор используемых блоков.

### `tests/test_scheme_method_selector.py`

Проверяет анализ схемы и выбор метода:

- пустая схема → `AUTO.IDENTITY`;
- один блок → `AUTO.SINGLE`;
- последовательная схема → F1.1;
- параллельная схема → композиционный метод с F1.2 как кандидатом;
- смешанная схема → композиционный метод;
- резервирование → F1.2;
- неиспользованный блок → предупреждение, но не включение в формулу.

### `tests/test_scheme_storage.py`

Проверяет сохранение и загрузку схемы.

### `tests/test_report_exporters.py`

Проверяет HTML, DOCX и XLSX smoke tests, наличие ключевых секций отчета и обязательных листов Excel.

### `tests/test_intelligent_formula_generator.py`

Проверяет интеллектуальный слой генератора формул:

- выбор формулы для последовательного фрагмента;
- выбор формулы для параллельного фрагмента;
- выбор формулы для MVP-резервирования;
- рекурсивную обработку вложенной схемы;
- наличие разделов выбора формул в UI-объекте `FormulaInfo`.

### `tests/test_validators.py`

Проверяет ошибки схемы: отсутствие входа/выхода, корректная простая структура.

### `tests/test_normative_methods.py`

Проверяет полноту реестра методов, наличие описаний, параметров и ограничений.

## 3. Встроенная пользовательская самопроверка

В графическом редакторе нажмите `Проверить систему`. Программа проверит встроенные шаблоны:

- последовательная схема;
- параллельная схема;
- комбинированная схема;
- резервирование 1+1.

В результате отображается:

- сколько примеров проверено;
- сколько успешно;
- какие блоки включены;
- какие блоки исключены;
- какой метод выбран;
- какая формула получена.

## 4. Ручная проверка UI

Перед демонстрацией рекомендуется проверить:

1. Запуск `python gui_main.py`.
2. Переключение между редактором и калькулятором.
3. Применение каждого шаблона.
4. Перемещение блоков и обновление линий.
5. Масштаб `−`, `+`, `100%`, `По схеме`.
6. Генерацию формулы в кратком и развернутом режиме.
7. Расчет схемы.
8. Экспорт PNG и SVG.
9. Сохранение и загрузку JSON.
10. Расчет в калькуляторе F1.1, F1.2, F2.2, F6.3, F7.2.
11. Экспорт HTML, DOCX, XLSX.

## 5. Проверка формул вручную

Последовательная схема из двух блоков:

```text
Pсист(t)=B1·B2
```

Параллельная схема из двух ветвей:

```text
Pсист(t)=1-(1-B1)(1-B2)
```

Резервирование 1+1:

```text
Pсист(t)=1-(1-R)^2
```

Если формула содержит `(B - 1)` вместо `(1 - B)` или `+1` поднят на верхний уровень всей системы, это регрессия генератора.

## 6. Проверка экспорта

Для каждого формата убедитесь, что:

- файл создается по выбранному пользователем пути;
- расширение корректное;
- отмена диалога не вызывает ошибку;
- отчет содержит метод, формулу и результаты;
- XLSX содержит листы `Сводка`, `Исходные данные`, `Результаты`, `Формулы`, `Данные графика`, `Графики`, `Заключение`; лист `Схема` добавляется при наличии изображения схемы.

## 7. Что делать при падении тестов

- Ошибка формулы: смотреть `graph_formula_builder.py`.
- Ошибка выбора метода: смотреть `scheme_method_selector.py`.
- Ошибка калькулятора: смотреть `normative_methods.py`, `method_metadata.py`, `dependability_backend.py`.
- Ошибка UI: смотреть соответствующий `gui_*.py`.
- Ошибка экспорта: смотреть `report_exporters.py` и `file_service.py`.
