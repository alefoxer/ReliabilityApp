# Архитектура проекта

## 1. Общая схема

Проект построен как desktop-приложение PyQt6 с плоской структурой модулей. Основной принцип: UI собирает данные и показывает результат, а расчет, формулы, выбор методов, сериализация и экспорт находятся в отдельных модулях.

Подробное описание каждого файла проекта вынесено в [MODULE_REFERENCE.md](MODULE_REFERENCE.md). В этом документе архитектура описывается на уровне подсистем и потоков данных, а справочник модулей раскрывает конкретные классы, функции и связи файлов.

```text
Пользователь
→ gui_main.py
→ gui_visual_editor.py или gui_calculator.py
→ rbd_models.py
→ validators.py
→ graph_formula_builder.py
→ formula_library.py / intelligent_formula_generator.py
→ scheme_method_selector.py
→ scheme_adapter.py / dependability_backend.py
→ report_exporters.py
```

## 2. Подсистемы

### Главное окно

`gui_main.py` создает:

- главное окно;
- меню;
- статус-бар;
- левую навигацию;
- стек рабочих модулей;
- подключение темы;
- связь редактора с калькулятором.

### Графический редактор

`gui_visual_editor.py` отвечает за:

- холст `QGraphicsScene/QGraphicsView`;
- блоки `RBDBlock`;
- связи `ConnectionLine`;
- панель шаблонов и файлов;
- расчет по схеме;
- генерацию общей формулы;
- встроенную самопроверку.

Графические элементы не являются единственным источником данных. Перед расчетом редактор собирает сериализуемую `SchemeModel`.

### Калькулятор

`gui_calculator.py` отвечает за ручной расчет по методам F1.1-F7.2. Он получает описание методов из `normative_methods.py`, строит форму ввода, вызывает backend-функцию и отображает результат.

### Модель данных

`rbd_models.py` содержит единые dataclass-модели. Это центральный контракт между UI, формулами, расчетом, экспортом и тестами.

### Генератор формул

`graph_formula_builder.py` строит формулу не строкой, а через AST:

- `FormulaExpr` — узел выражения;
- `NormalizedScheme` — нормализованная расчетная схема;
- `FormulaGenerationResult` — полный результат генерации;
- `FormulaInfo` — объект для UI и отчетов.

`formula_library.py` содержит библиотеку проектных формул для типовых фрагментов схемы: единичный элемент, последовательное соединение, параллельное соединение, MVP-резервирование и нейтральная пустая схема. Формулы помечены как методические формулы проекта, если для них в материалах проекта нет отдельной ГОСТ-ссылки.

`intelligent_formula_generator.py` дополняет базовый AST-генератор интеллектуальным слоем: проходит по дереву формулы, определяет тип каждого фрагмента, выбирает подходящую запись из библиотеки, подставляет число элементов, резервов и параметры блоков, а затем добавляет в `FormulaInfo` разделы с выбранными формулами, подстановками и численным результатом.

### Выбор метода

`scheme_method_selector.py` выполняет интеллектуальный слой:

- анализирует признаки схемы;
- определяет сценарии;
- формирует кандидаты методов;
- выбирает рекомендуемый метод;
- объясняет выбор.

### Расчет

Есть два уровня расчета:

- `dependability_backend.py` — нормативные/проектные функции F1.1-F7.2;
- `scheme_adapter.py` — расчет по построенной схеме с учетом формулы и выбора метода.

### Экспорт

`report_exporters.py` принимает `ReportData` и сохраняет документы. UI выбирает путь через `file_service.py`, а экспортеры только записывают данные.

## 3. Поток данных “схема → формула → расчет”

1. Пользователь строит схему.
2. `ModuleVisualRBD.to_scheme_model()` формирует `SchemeModel`.
3. `validate_scheme()` проверяет структуру.
4. `build_formula_report()` нормализует схему.
5. Из достижимого подграфа строится AST.
6. Из AST строятся:
   - символьная формула;
   - вычислительная формула;
   - структурное представление;
   - обозначения;
   - объяснение;
   - предупреждения.
7. `enhance_formula_info()` сопоставляет фрагменты AST с библиотекой формул и добавляет объяснение выбора формул.
8. `select_method_for_scheme()` анализирует результат и рекомендует метод.
9. `calculate_scheme_reliability()` рассчитывает показатели.
10. `CalculationResult` отображается в UI и передается в калькулятор.

## 4. Нормализация схемы

Нормализация нужна, чтобы все представления были согласованы. Генератор выделяет:

- вход;
- выход;
- расчетные блоки;
- связи;
- достижимые элементы от входа к выходу;
- неиспользованные блоки;
- предупреждения.

Если блок есть на canvas, но не входит в путь от входа к выходу, он не попадает в формулу, но указывается в предупреждениях.

## 5. AST формулы

Минимальные типы выражений:

- `one` — константа 1;
- `symbol` — элемент схемы;
- `series` — последовательное соединение;
- `parallel` — параллельное соединение;
- `reserve` — резервирование MVP.

AST используется как единый источник истины для всех разделов:

- итоговая формула;
- вычислительное представление;
- обозначения;
- объяснение;
- тесты согласованности.

## 6. Композиционные правила

Последовательное соединение:

```text
P = P1 · P2 · ... · Pn
Kг = K1 · K2 · ... · Kn
```

Параллельное соединение:

```text
P = 1 - (1 - P1)(1 - P2)...(1 - Pn)
Kг = 1 - (1 - K1)(1 - K2)...(1 - Kn)
```

Резервирование MVP:

```text
P = 1 - (1 - Pэлемента)^(m+1)
```

## 7. Файловые форматы

### JSON схемы

JSON хранит не `QGraphicsItem`, а чистую модель:

- блоки;
- связи;
- параметры;
- координаты;
- вложенные схемы;
- metadata.

### Отчеты

`ReportData` содержит:

- входные данные;
- результаты;
- методику;
- формулу;
- таблицы;
- графики;
- путь к изображению схемы;
- примечания.

## 8. Ограничения архитектуры

- UI и расчет еще находятся в одной файловой структуре без пакетов `core/` и `ui/`, чтобы сохранить совместимость текущего запуска.
- Сложные циклические графы не поддерживаются генератором формул.
- Тяжелые расчеты пока выполняются синхронно.
- DOCX экспорт реализован через `python-docx`, XLSX экспорт через `openpyxl`, PDF экспорт через Qt. Для сдачи этого достаточно, но промышленный документооборот может потребовать отдельного шаблонизатора и более строгого рендера формул.

## 9. Точки расширения

- Вынос тяжелых расчетов в worker/thread.
- Расширение типов AST.
- Добавление новых нормативных методов.
- Более глубокая поддержка иерархических схем.
- Расширение библиотеки шаблонов по ОСТ.
- Улучшение рендера формул в DOCX.
