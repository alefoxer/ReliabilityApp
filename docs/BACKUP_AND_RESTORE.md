# Backup и восстановление файлов

В проекте действует обязательное правило: перед изменением любого существующего файла сначала сохраняется его исходная копия в `backup/`.

## Как создаются backup-копии

- Корневая папка для копий: `backup/`
- Для каждой задачи используется отдельная подпапка:
  - `backup/YYYY-MM-DD_краткое-название-патча/`
- В backup попадают только **существующие файлы**, которые будут изменяться.
- Для **новых** файлов backup не требуется.
- Если один и тот же файл правится несколько раз в рамках одной задачи, backup делается один раз: до первой правки.

## Что считается последней версией для отката

Последней версией считается backup-копия, сохранённая **перед текущей задачей**.

- Если сломался один файл, восстанавливается только он.
- Если проблема появилась в группе связанных правок, восстанавливаются все файлы из одной backup-папки патча.

## Порядок восстановления

1. Найти последнюю подходящую папку в `backup/`.
2. Определить, какие файлы нужно вернуть.
3. Восстановить их из backup-копий.
4. Явно зафиксировать, какие файлы были восстановлены и из какой backup-папки.

При необходимости текущую проблемную версию тоже можно отдельно сохранить перед откатом, чтобы не потерять промежуточное состояние.

## Что не входит в это правило по умолчанию

По умолчанию правило не распространяется на автоматически создаваемые артефакты, если они не редактируются вручную:

- `.venv`
- `build`
- `dist`
- `__pycache__`
- `.pytest_cache`
- временные выходные файлы и логи

Если такой файл или папка редактируется вручную как часть задачи, backup можно сделать отдельно по тому же принципу.
