# Справочник интерфейса

## 1. Главное окно

### Левая колонка

`Модули`:

- `Графический редактор` — открывает построение схем.
- `Калькулятор` — открывает ручной расчет по методам.

При выборе графического редактора под блоком `Модули` показываются `Шаблоны и файлы` и `Общая формула системы`.

### Верхнее меню

Меню используется для действий приложения: расчет, формула, отчет, экспорт, справка и масштаб интерфейса. Конкретные действия дублируют кнопки модулей, чтобы пользователь мог работать и через меню, и через панели.

### Статус-бар

Показывает короткие статусы:

- схема загружена;
- расчет выполнен;
- отчет сохранен;
- ошибка экспорта;
- готовность приложения.

## 2. Графический редактор

### Левая панель редактора

`Шаблоны и файлы`:

- выпадающий список шаблонов — выбор демонстрационной схемы;
- `Применить шаблон` — очищает текущую схему и загружает выбранный шаблон;
- `Проверить схему` — запускает валидацию;
- `Сохранить схему` — сохраняет JSON;
- `Загрузить схему` — открывает JSON;
- `Экспорт SVG` — сохраняет схему как SVG;
- `Экспорт PNG` — сохраняет схему как PNG;
- `Очистить` — удаляет текущие элементы сцены.

`Общая формула системы`:

- показывает имя схемы;
- количество блоков и связей;
- результат проверки;
- анализ структуры;
- рекомендованный метод;
- формулу;
- предупреждения;
- последний расчет.

### Центральная доска

Действия:

- перетаскивание блока — перемещает блок, линии обновляются автоматически;
- выделение блока — показывает стандартное выделение;
- контекстное меню блока — редактирование имени, параметров, подсхемы или удаление;
- создание связи — выполняется через порты/механизм редактора;
- колесо мыши с `Ctrl` — масштаб;
- средняя кнопка мыши — панорамирование.

### Правая панель редактора

`Добавление блоков`:

- `Вход` — добавляет стартовый узел;
- `Выход` — добавляет конечный узел;
- `Горизонтальный блок` — добавляет основной функциональный блок;
- `Вертикальный блок` — добавляет блок для вертикальной компоновки;
- `Разветвление вверх` — добавляет элемент ветвления;
- `Разветвление вниз` — добавляет элемент ветвления/компоновки вниз.

`Параметры расчета схемы`:

- `Метод` — выбор способа расчета по схеме;
- `Горизонт t` — время оценки;
- `Число симуляций` — объем Monte Carlo;
- `Рассчитать схему` — запускает расчет.

`Генератор общей формулы`:

- `Краткий вид` — показывает компактный фрагмент формулы;
- `Развернутый вид` — показывает формулу, структуру, обозначения и шаги;
- `Сгенерировать формулу` — открывает окно формулы;
- `Копировать формулу` — помещает формулу и объяснение в буфер обмена;
- `Проверить систему` — запускает встроенную самопроверку на шаблонах.

`Масштаб рабочей области`:

- `−` — уменьшить масштаб;
- `100%` — сбросить масштаб;
- `+` — увеличить масштаб;
- `По схеме` — подогнать вид к содержимому;
- слайдер — ручной выбор масштаба.

## 3. Контекстное меню блока

В зависимости от блока доступны действия:

- переименовать;
- изменить параметры;
- исследовать как подсхему;
- удалить.

`Исследовать как подсхему` означает, что блок становится контейнером внутренней структуры. Его вложенная формула учитывается при расчете внешней схемы.

## 4. Калькулятор

### Выбор метода

В верхней части калькулятора находится список методов F1.1-F7.2. После выбора метода форма параметров перестраивается автоматически.

### Подробности метода

Раскрываемый блок `Подробности метода` по умолчанию свернут. Внутри показываются:

- название;
- описание;
- где применяется;
- что рассчитывает;
- сценарии;
- расшифровка параметров;
- формулы;
- ограничения.

### Форма параметров

В форме ввода показываются только краткие обозначения параметров и поля значений. Длинная расшифровка находится в подробностях метода и tooltips.

### Таблица элементов

Для векторных методов используется таблица элементов. В нее вводятся параметры по каждому элементу, например `λi`, `Tvi`, `T0i`.

### Кнопки калькулятора

- `Рассчитать` — выполняет выбранную функцию backend.
- `Экспорт отчёта` — сохраняет отчет в выбранном формате.
- `Экспорт графика` — сохраняет график.
- `Справочник` — открывает справочную/шаблонную информацию.
- `Параметры отчёта` — задает название, изделие, примечания.
- `Сохранить шаблон` — сохраняет текущие данные как шаблон компонента.
- `Загрузить шаблон` — загружает сохраненный шаблон.

## 5. Диалоги файлов

Для всех экспортов пользователь выбирает путь и имя файла. Если действие отменено, приложение не падает и просто возвращается к работе.

## 6. Диалоги ошибок

Ошибки показываются обычным пользователю языком. Технические детали пишутся в лог.

## 7. Финальная демонстрация UI

Перед показом преподавателю проверьте чеклист [FINAL_READINESS.md](FINAL_READINESS.md): название окна, отсутствие технических подписей, переключение модулей, компактный экран результатов, наличие графика, формулы под графиком и экспорт отчетов.
