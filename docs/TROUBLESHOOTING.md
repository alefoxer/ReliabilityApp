# Типовые проблемы, ошибки и ограничения

## 1. Приложение не запускается

Проверьте:

- установлен ли Python;
- доступна ли команда `python`;
- установлены ли зависимости из `requirements.txt`;
- запускается ли файл из корня проекта.

Команды:

```bash
pip install -r requirements.txt
python gui_main.py
```

## 2. Не видна формула после нажатия кнопки

Проверьте:

- есть ли на схеме вход и выход;
- соединены ли блоки в путь от входа к выходу;
- нет ли блоков, висящих отдельно;
- нет ли циклов;
- не свернут ли блок `Общая формула системы`.

Нажмите `Проверить схему` и затем `Сгенерировать формулу`.

## 3. Блок есть на схеме, но не попал в формулу

Это допустимо, если блок не находится на пути от входа к выходу. Программа должна вывести предупреждение в разделе формулы. Такой блок не включается в расчет, чтобы не искажать результат.

## 4. Линии выглядят оторванными

Линии должны быть объектно привязаны к блокам. Если после редактирования появляется рассинхронизация:

1. сохраните схему;
2. загрузите ее заново;
3. проверьте, что связи восстановились;
4. если проблема повторяется, смотреть `ConnectionLine` и `RBDBlock.itemChange()` в `gui_visual_editor.py`.

## 5. Экспорт PNG пустой

Экспорт должен использовать `scene.itemsBoundingRect()` и рендерить содержимое сцены, а не скриншот видимой области. Если PNG пустой:

- проверьте, есть ли элементы на сцене;
- проверьте `scene_exporters.py`;
- убедитесь, что путь сохранения выбран и доступен.

## 6. Отчет не сохраняется

Проверьте:

- выбран ли путь в диалоге;
- не открыт ли файл в другой программе;
- есть ли права записи в выбранную папку;
- корректно ли расширение.

Если пользователь отменил диалог, это не ошибка.

## 7. Метод в калькуляторе недоступен или результат кажется странным

Проверьте:

- выбранный сценарий;
- значения параметров;
- диапазоны `λ`, `t`, `n`, `m`;
- подробности метода;
- ограничения F6.3 по `T0`.

Сценарий внутри метода меняет формулу. Например F1.2 для нагруженного и ненагруженного резерва дает разные результаты.

## 8. Ограничения текущей версии

- Циклические графы не поддерживаются генератором формул.
- Генератор ориентирован на последовательно-параллельные структуры и MVP-резервирование.
- Вложенные схемы поддерживаются как первая рабочая версия.
- Monte Carlo пока выполняется синхронно.
- DOCX и XLSX создаются встроенными средствами и ориентированы на демонстрационный отчет.
- Для DOCX требуется зависимость `python-docx`, для XLSX требуется `openpyxl`, для вставки изображений в Excel используется `Pillow`. Все эти зависимости указаны в `requirements.txt`.

## 9. Где смотреть логи

Логи находятся в каталоге `logs/`. Пользователь видит дружелюбное сообщение, а технические детали должны попадать в лог.
