# Пример из PDF для СНЭ ЭМРТУ

Этот файл подготовлен по документу:

- `МКТН_563255_005_02_РР3_Система_накопелния_и_хранения_энергии_ЭМРТУ.pdf`

Назначение файла:

- описать готовый демонстрационный пример;
- показать, какие данные загружаются в программе;
- зафиксировать исходные данные;
- зафиксировать ожидаемые результаты;
- отдельно отметить, что в PDF есть расчеты, которые текущая версия программы не покрывает напрямую.

## 1. Расчетная область примера

В текущей версии приложения по PDF воспроизводится верхнеуровневая структурная схема СНЭ как последовательное соединение 6 подсистем:

1. Энергоблок `МКТН.563255.008`
2. Энергоблок собственных нужд `МКТН.563255.007`
3. Подсистема управления электропитанием `МКТН.794775.025`
4. Комплект силовых кабельных линий `МКТН.468921.082`
5. Комплект информационных кабельных линий `МКТН.468921.083`
6. Распределительное устройство проходных коробок `МКТН.674211.009`

Это соответствует разделам PDF:

- раздел `9.1.2` — последовательное соединение;
- таблица 4 — итоговые значения `P(t)`;
- таблица 5 — итоговые значения `Kг`.

## 2. Подготовленные файлы схемы

В PDF есть несогласованность чисел:

- таблица 4 содержит итог для СНЭ на `158 ч`: `0.998900209`;
- страница с выводами содержит значение `0.999992793` для `158 ч`;
- произведение значений таблицы 4 по строкам 1-6 дает именно `0.998900209`.

Поэтому таблица 4 используется как основной эталон.

Также значения `P(t1=158 ч)` и `P(t2=125 ч)` в таблице 4 не согласуются между собой как один и тот же набор экспоненциальных `lambda`. Поэтому подготовлены два отдельных файла:

- [pdf_snhe_158h_scheme.json](../examples/pdf_snhe_158h_scheme.json) — для точного воспроизведения таблицы 4 на `t = 158 ч`;
- [pdf_snhe_125h_scheme.json](../examples/pdf_snhe_125h_scheme.json) — для точного воспроизведения таблицы 4 на `t = 125 ч`.

В оба файла дополнительно включены значения `Kg` из таблицы 5, чтобы готовность считалась по тем же блочным данным, что и в PDF.

## 3. Сценарии загрузки в программе

### Вариант A. Проверка PDF на `158 ч`

1. Запустить программу.
2. Открыть модуль `Графический редактор`.
3. Использовать команду `Загрузить схему`.
4. Выбрать файл [pdf_snhe_158h_scheme.json](../examples/pdf_snhe_158h_scheme.json).
5. В правой панели в блоке `Параметры расчета схемы` установить:
   - `Метод`: `Аналитический расчёт`
   - `Горизонт t`: `158`
6. Выполнить действие `Проверить схему`.
7. Выполнить действие `Сгенерировать формулу`.
8. Выполнить действие `Рассчитать схему`.

Ожидаемый смысл формулы:

```text
Pсист(t) = B1 · B2 · B3 · B4 · B5 · B6
Kг_сист = K_B1 · K_B2 · K_B3 · K_B4 · K_B5 · K_B6
```

### Вариант B. Проверка PDF на `125 ч`

1. Запустить программу.
2. Открыть модуль `Графический редактор`.
3. Использовать команду `Загрузить схему`.
4. Выбрать файл [pdf_snhe_125h_scheme.json](../examples/pdf_snhe_125h_scheme.json).
5. В правой панели в блоке `Параметры расчета схемы` установить:
   - `Метод`: `Аналитический расчёт`
   - `Горизонт t`: `125`
6. Выполнить действие `Проверить схему`.
7. Выполнить действие `Сгенерировать формулу`.
8. Выполнить действие `Рассчитать схему`.

| `P(B2)` | `0.999918418` |
| `P(B3)` | `0.999248387` |
| `P(B4)` | `0.999829933` |
| `P(B5)` | `0.999914963` |
| `P(B6)` | `0.999999500` |

- в итоговой строке PDF указано `0.99999982`, поэтому здесь есть небольшая несостыковка самого исходного документа.

## 7. Какие Части PDF Наша Программа Не Покрывает Напрямую

Следующие части PDF не стоит выдавать за уже полностью автоматизированные в текущей версии:

- раздел `9.2` — функциональная надежность с резервом времени `tз`, `tи`;
- раздел `11` — долговечность, средний ресурс, срок службы, общий срок службы;
- часть расчетов по восстановлению и резервированию в специальных обозначениях PDF.

Корректная формулировка ограничения:

`В текущей версии программы полностью воспроизводится структурная проверка верхнеуровневой схемы СНЭ по вероятности безотказной работы и коэффициенту готовности. Специальные модели функциональной надежности и долговечности из данного PDF пока не вынесены в отдельные автоматические методики интерфейса.`
