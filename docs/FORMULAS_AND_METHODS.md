# Формулы, методики и параметры

## 1. Два вида формул в проекте

В проекте есть два связанных, но разных уровня формул.

### Общая формула по структурной схеме

Эта формула строится автоматически по графической схеме. Она показывает, как надежность всей системы получается из надежности отдельных блоков.

Пример:

```text
Pсист(t) = B1 · B2
Kг_сист = K_B1 · K_B2
```

или для параллельной схемы:

```text
Pсист(t) = 1 - (1 - B1)(1 - B2)
```

### Формулы методик F1.1-F7.2

Эти формулы используются в калькуляторе. Они заданы в `dependability_backend.py` и описаны в `normative_methods.py`. Пользователь выбирает метод, сценарий и параметры, а программа выполняет расчет.

### Единый рендер формул

Отображение формул в интерфейсе и отчетах выполняется через общий модуль `formula_rendering.py`. Он переводит LaTeX-подобные строки из реестра методик в человекочитаемый вид: `\frac{a}{b}` показывается как `(a) / (b)`, `\lambda` как `λ`, `\sum` как `Σ`, `\prod` как `Π`. Это сделано, чтобы в калькуляторе, HTML, DOCX, XLSX и текстовом отчете не появлялись технические команды или слова вроде “дробь”.

Для выводимых показателей дополнительно задан общий каталог формул:

- `P(t)` — вероятность безотказной работы за время `t`;
- `T0` — средняя наработка до отказа;
- `Kг = T0 / (T0 + Tв)` — коэффициент готовности;
- `Kог = Kг · P(t)` — коэффициент оперативной готовности;
- `Tв` — среднее время восстановления;
- `Tпр` — среднее время простоя;
- `λ = 1 / T0` — интенсивность отказов при использовании обратной оценки через наработку.

Если для методики показатель не подтвержден текущим набором материалов, он не должен выводиться как полноценный результат. Например, для F6.3 в интерфейсе оставлен расчет `P(t)`, а `T0` не показывается как рабочий показатель.

## 2. Генератор общей формулы

Генератор работает через AST:

```text
SchemeModel → NormalizedScheme → FormulaExpr → FormulaInfo
```

Он формирует:

- структурное представление;
- символьную формулу;
- вычислительное представление;
- обозначения;
- пошаговое объяснение;
- список включенных и исключенных блоков;
- предупреждения.

### Интеллектуальный слой выбора формул

После построения AST результат дополняется модулем `intelligent_formula_generator.py`. Он не подменяет базовую формулу, а объясняет, какие типовые формулы из `formula_library.py` применены к каждому фрагменту схемы.

Конвейер:

```text
FormulaExpr → фрагменты схемы → FormulaDefinition → подстановка N, m, λ, Tв → объяснение → FormulaInfo
```

Библиотека `formula_library.py` содержит проектные формулы для:

- пустой схемы: `Pсист(t)=1`, `Kг_сист=1`;
- простого элемента: `Pэ(t)=exp(-λэ·t)`;
- последовательного соединения: `P=Π Pi`;
- параллельного соединения: `P=1-Π(1-Pi)`;
- MVP-резервирования: `P=1-(1-Pэ)^(m+1)`;
- коэффициента готовности для последовательных и параллельных групп.

Если формула не имеет отдельной подтвержденной ГОСТ-ссылки в материалах проекта, она обозначается как “методическая формула проекта”. Это сделано намеренно: система не приписывает формулам ложную нормативную базу.

В окне формулы пользователь видит не только итоговую строку, но и:

- выбранные формулы по фрагментам;
- количество элементов `N`;
- параметры резервирования `m`;
- подставленные значения `λ`, `Tв`, `t`;
- численный результат, если параметры заданы.

## 3. Правила композиции

### Последовательное соединение

Система работает, если работают все элементы:

```text
P = P1 · P2 · ... · Pn
Kг = K1 · K2 · ... · Kn
```

### Параллельное соединение

Система работает, если работает хотя бы одна ветвь:

```text
P = 1 - (1 - P1)(1 - P2)...(1 - Pn)
Kг = 1 - (1 - K1)(1 - K2)...(1 - Kn)
```

### Вложенная схема

Если блок помечен как подсхема, формула внутренней схемы строится рекурсивно и подставляется в формулу внешней схемы.

### Резервирование MVP

Если у блока задан `reserve_count=m`, используется композиционная модель:

```text
P = 1 - (1 - Pэлемента)^(m+1)
```

## 4. Основные обозначения

- `P(t)` — вероятность безотказной работы на интервале времени `t`.
- `Pсист(t)` — вероятность безотказной работы всей системы.
- `T0` — средняя наработка до отказа.
- `Tv` — среднее время восстановления.
- `Kg` — коэффициент готовности.
- `Kog` — коэффициент оперативной готовности.
- `λ` — интенсивность отказов.
- `λp` — интенсивность отказов в облегченном резерве.
- `λs` — интенсивность отказов переключателя.
- `n` — число основных элементов.
- `m` — число резервных элементов или допустимое число отказов.
- `r1`, `r2`, `r3` — размеры уровней в структурных/мажоритарных методах.
- `γ` — доля обнаруживаемых отказов.
- `Ps` — вероятность успешного переключения или срабатывания.

## 5. Сценарии расчета

Сценарий — это вариант применения метода внутри выбранной методики. Например, F1.2 содержит несколько сценариев резервирования:

- нагруженный резерв;
- облегченный резерв;
- ненагруженный резерв;
- неоднородные элементы.

Сценарий меняет формулу и смысл параметров. Поэтому результаты для сценария 1 и сценария 3 могут отличаться даже при похожих `n`, `m`, `λ` и `t`.

## 6. Реестр методов

Ниже приведен практический справочник методик, реализованных в проекте. Формулы указаны в человекочитаемом сокращенном виде; точная вычислительная реализация находится в `dependability_backend.py`, а UI берет формулы из `normative_methods.py`.

### F1.1 Последовательное соединение

- Применяется: независимые невосстанавливаемые элементы, соединенные последовательно.
- Рассчитывает: `P(t)`, `T0`.
- Параметры: `t`, таблица `λi`.
- Формулы:

```text
P(t)=exp(-t · Σλi)
T0=1/Σλi
```

### F1.2 Сложное резервирование

- Применяется: нагруженный, облегченный, ненагруженный или неоднородный резерв.
- Рассчитывает: `P(t)`, `T0`.
- Параметры: `cat3`, `t`, `n`, `m`, `λ`, `λp`, для неоднородного случая список `λi`.
- Формулы:

```text
cat3=1: P(t)=Σ C_N^i exp(-(N-i)λt)(1-exp(-λt))^i
cat3=2: используется отношение a=λp/λ и сумма по резервным состояниям
cat3=3: P(t)=exp(-nλt) · Σ((nλt)^i/i!)
T0(cat3=3)=(m+1)/(nλ)
```

### F1.3 Дублирование с переключателем

- Применяется: дублированная структура с ненадежным переключателем.
- Рассчитывает: `P(t)`, `T0`.
- Параметры: `t`, `λ`, `λs`.
- Формулы используют экспоненциальные слагаемые для отказов элементов и переключателя.

### F1.4 Мажоритарная структура 2 из 3

- Применяется: схема, где работоспособность сохраняется при наличии двух исправных элементов из трех.
- Рассчитывает: `P(t)`, `T0`.
- Параметры: `t`, `λ1`, `λ2`.
- Формулы:

```text
P(t)=(exp(-3λ1t)+3exp(-2λ1t)(1-exp(-λ1t)))exp(-λ2t)
T0=3/(2λ1+λ2)-2/(3λ1+λ2)
```

### F1.5 Две мажоритарные группы

- Применяется: две последовательно учитываемые мажоритарные группы.
- Рассчитывает: `P(t)`, `T0`.
- Параметры: `t`, `λ1`, `λ2`.
- Формула `P(t)` равна произведению надежностей двух мажоритарных групп.

### F2.1 Восстанавливаемая система

- Применяется: восстанавливаемые системы с независимым или одновременным восстановлением.
- Рассчитывает: `P`, `T0`, `Tv`, `Kg`, `Kog`.
- Параметры: `cat3_f2`, `t`, `n`, таблицы `λi`, `Tvi`, `T0i`.
- Формулы:

```text
Kg=Π 1/(1+λiTvi)
P(t)=Π exp(-t/T0i)
```

### F2.2 Резерв с восстановлением

- Применяется: резервированные системы с восстановлением.
- Рассчитывает: `P`, `T0`, `Tv`, `Kg`, `Kog`.
- Параметры: `cat3_f22`, `t`, `n`, `m`, `Tv`, `λ`, `λp`, `λs`.
- Сценарии: нагруженный, облегченный, ненагруженный, ненадежный переключатель.

### F2.3 Мажоритарная восстанавливаемая система

- Применяется: восстанавливаемые структуры m из n.
- Рассчитывает: `P`, `T0`, `Tv`, `Kg`, `Kog`.
- Параметры: `t`, `m`, таблицы `λi`, `Tvi`.
- Формулы используют произведения `λiTvi` и коэффициент готовности через вероятность отказа всех критичных элементов.

### F2.4 Неполный контроль

- Применяется: системы, где часть отказов обнаруживается контролем.
- Рассчитывает: `P`, `T0`, `Tv`, `Kg`, `Kog`.
- Параметры: `cat3_f24`, `t`, `n`, `m`, `λ`, `Tv`, `Tobn`, `γ`, `λs`.

### F2.5 Смешанный контроль / переключение

- Применяется: системы с контролем, переключением и вероятностью успешного срабатывания.
- Рассчитывает: `P`, `T0`, `Tv`, `Kg`, `Kog`.
- Параметры: `cat3`, `t`, `n`, `λ`, `Tv`, `Tobn`, `Tvp`, `γ`, `Ps`, `λp`.

### F2.6 Два разных элемента с восстановлением

- Применяется: система с двумя различными восстанавливаемыми элементами.
- Рассчитывает: `P`, `T0`, `Tv`, `Kg`, `Kog`.
- Параметры: `t`, `λ1`, `λ2`, `Tv1`, `Tv2`.

### F2.7 Два симметричных элемента

- Применяется: двухэлементная симметричная восстанавливаемая структура.
- Рассчитывает: `P`, `T0`, `Tv`, `Kg`, `Kog`.
- Параметры: `t`, `λ1`, `λ2`, `Tv1`, `Tv2`.

### F3.1 Периодический контроль

- Применяется: структуры m из N с периодическим контролем.
- Рассчитывает: `P`, `T0`, `Tv`, `Kg`, `Kog`.
- Параметры: `cat3`, `t`, `n`, `m`, `λ`, `Tp`, `Tv`.

### F4.1 Смешанный m из n с контролем

- Применяется: смешанная структура с контролем и двумя типами интенсивностей отказов.
- Рассчитывает: `P`, `T0`, `Tv`, `Kg`, `Kog`.
- Параметры: `t`, `n`, `m`, `λ1`, `λ2`, `Tv2`, `Tp`.

### F5.1 Допустимое время простоя

- Применяется: расчет с учетом допустимого времени простоя.
- Рассчитывает: `P`, `T0`, `Tpr`, `Kg`, `Kog`.
- Параметры: `t`, `λ`, `tdop`, `Tv_in`.
- Формулы:

```text
T0=exp(tdop/Tv_in)/λ
Tpr=Tv_in · exp(-tdop/Tv_in)
Kg=T0/(T0+Tpr)
```

### F6.1 Мажоритарная структура m из r1

- Применяется: структурная надежность с управляющим элементом.
- Рассчитывает: `P(t)`, `T0`.
- Параметры: `t`, `r1`, `m`, `λupr`, `λ`.

### F6.2 Двухуровневая мажоритарность

- Применяется: двухуровневые мажоритарные структуры.
- Рассчитывает: `P(t)`, `T0`.
- Параметры: `t`, `r1`, `r2`, `m`, `λ1`, `λ2`, `Tupr`.

### F6.3 Трехуровневая мажоритарность

- Применяется: трехуровневые мажоритарные структуры.
- Рассчитывает: `P(t)`.
- Параметры: `t`, `r1`, `r2`, `r3`, `m`, `λ1`, `λ2`, `λ3`, `Tupr`.
- Ограничение: в текущем утвержденном наборе материалов `T0` не реализован и возвращается как `0.0`; это явно документировано.

### F7.1 Структура с покрытием

- Применяется: структуры с покрытием отказов.
- Рассчитывает: `T0`, `Kg`, `Kog`.
- Параметры: `r1`, `m`, `Tupr`, `Kupr`, `Koupr`, `T1`, `K1`, `Ko1`.

### F7.2 Двухуровневая структура с покрытием

- Применяется: двухуровневые структуры с покрытием отказов.
- Рассчитывает: `T0`, `Kg`, `Kog`.
- Параметры: `r1`, `r2`, `m`, `Tupr`, `Kupr`, `Koupr`, `T1`, `K1`, `Ko1`, `T2`, `K2`, `Ko2`.
