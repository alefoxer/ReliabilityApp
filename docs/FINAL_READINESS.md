# Финальная готовность проекта к сдаче

Этот документ нужен как короткий контрольный лист перед демонстрацией преподавателю. Он фиксирует, что проверять вручную, какие тестовые схемы использовать и какие автоматические тесты должны проходить.

## 1. Быстрый статус

- Рабочее название приложения: `Надёжность технических средств`.
- Основные модули: графический редактор, калькулятор, генератор формул, отчеты.
- Основной запуск: `python gui_main.py`.
- Полная проверка: `python -m pytest`.
- На момент финальной проверки автоматический набор содержит 57 тестов.

## 2. End-to-end сценарий демонстрации

1. Запустить приложение из корня проекта.
2. Проверить, что в заголовке указано `Надёжность технических средств` и нет слова `Python`.
3. Переключиться между модулями `Графический редактор` и `Калькулятор`.
4. В графическом редакторе открыть шаблон последовательной схемы.
5. Нажать `Проверить схему`.
6. Нажать `Сгенерировать формулу`.
7. Проверить, что формула содержит используемые блоки и объяснение выбора формул.
8. Нажать `Рассчитать схему`.
9. Открыть `Показать результаты`.
10. Проверить числовую таблицу результатов, график и формулу под графиком.
11. Экспортировать отчет в HTML.
12. Экспортировать отчет в DOCX.
13. Экспортировать отчет в XLSX.
14. Сохранить схему в JSON.
15. Загрузить сохраненную схему и убедиться, что блоки и связи восстановились.
16. Нажать `Проверить систему` для встроенной самопроверки генератора.

## 3. Тестовые схемы

| Схема | Ожидаемый тип | Ожидаемая формула | Ожидаемый результат | Предупреждения |
| --- | --- | --- | --- | --- |
| Пустая схема | Нейтральная схема | `Pсист(t)=1`, `Kг_сист=1` | Расчет допустим как граничный случай | Нет |
| Один блок | Единичный элемент | `Pсист(t)=B1` | Значение равно надежности блока | Нет |
| Последовательная схема | Последовательное соединение | `Pсист(t)=B1 · B2 · ... · Bn` | Результат уменьшается при добавлении элементов | Нет |
| Параллельная схема | Параллельное соединение | `Pсист(t)=1 - Π(1 - Bi)` | Результат выше надежности одиночной ветви при независимости | Нет |
| Смешанная схема | Последовательно-параллельная композиция | Рекурсивная комбинация `series` и `parallel` | Расчет совпадает с AST-формулой | Нет |
| Резервирование 1+1 | MVP-резервирование | `P=1-(1-Pэ)^2` | Результат выше одиночного элемента | Ограничение MVP-модели |
| Вложенная схема | Подсхема внутри блока | Внутренняя формула подставляется во внешнюю | Расчет выполняется рекурсивно | Нет при корректной подсхеме |
| Неподключенный блок | Некорректная/частичная схема | Блок не входит в итоговую формулу | Расчет по достижимому подграфу или блокировка по валидатору | Предупреждение о неиспользуемом блоке |
| Большая схема | Масштабируемая композиция | Формула строится по AST без ручных шаблонов | Тест должен завершиться без падения | Возможна длинная формула |

## 4. Проверка интерфейса

- Заголовок окна: `Надёжность технических средств`.
- Нет слова `Python` и временных технических подписей.
- Кнопка закрытия в окне формулы подписана `Закрыть`.
- Левая панель содержит модули, шаблоны/файлы и общую формулу системы.
- Правая панель содержит действия, параметры расчета, генерацию формулы и масштаб.
- Экран результатов содержит числовую таблицу, график, формулу и вывод по порогу.
- Калькулятор показывает компактную форму ввода и раскрываемое описание методики.

## 5. Проверка формул

- В формулах нет словесных замен вроде `дробь`.
- Последовательное соединение отображается как произведение.
- Параллельное соединение отображается как `1 - Π(1 - Pi)`.
- `+1` не выносится ошибочно на верхний уровень всей системы.
- Обозначения строятся из AST и совпадают с блоками формулы.
- Неиспользуемые блоки отображаются отдельно в предупреждениях.
- Под графиком калькулятора выводится формула выбранной методики.

## 6. Проверка экспорта

HTML:

- открывается локально;
- содержит титульный блок, методику, формулы, результаты, график, таблицы и заключение.

DOCX:

- открывается в Word или совместимом редакторе;
- содержит заголовки, таблицу результатов, формулы и график;
- не содержит сырых технических строк.

XLSX:

- открывается в Excel или совместимом редакторе;
- содержит листы `Сводка`, `Исходные данные`, `Результаты`, `Формулы`, `Данные графика`, `Графики`, `Заключение`;
- содержит числовые данные графика и итог по порогу.

## 7. Автоматические тесты

Запуск:

```bash
python -m pytest
```

Покрытие:

- расчетные функции F1.1-F7.2;
- генератор формул и AST-инварианты;
- интеллектуальный выбор формул из библиотеки;
- выбор сценария и метода по схеме;
- сериализация и загрузка схем;
- валидация схем;
- экспорт HTML, DOCX и XLSX.

## 8. Ограничения, которые важно честно озвучить

- Генератор ориентирован на ациклические структурные схемы с одним входом и одним выходом.
- Резервирование в графическом редакторе реализовано как рабочий MVP.
- Для части проектных композиционных формул источник указан как `Методическая формула проекта`, если отдельная ГОСТ-ссылка не задана в материалах.
- Сложные циклические графы и промышленные офисные шаблоны отчетов не являются целью текущего прототипа.

## 9. Что показать преподавателю

1. Построение схемы из шаблона.
2. Проверку схемы и предупреждение по неподключенному блоку.
3. Генерацию общей формулы и объяснение выбора формул.
4. Числовой расчет и график.
5. Экспорт DOCX и XLSX.
6. Встроенную самопроверку генератора.
7. Документацию: README, USER_GUIDE, FORMULAS_AND_METHODS, MODULE_REFERENCE и этот чеклист.
