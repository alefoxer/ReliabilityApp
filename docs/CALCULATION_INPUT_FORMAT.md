# Формат входных JSON/YAML файлов для расчетов

Проект поддерживает два близких, но разных формата входных файлов.

1. Внутренний файл схемы редактора, который читает `scheme_storage.py`.
2. Нормализованный импорт инженерного расчета, который читает `external_reliability_import.py`.

Примеры лежат в `examples/`:

- `examples/pdf_snhe_125h_scheme.json` и `examples/pdf_snhe_158h_scheme.json` - внутренние схемы редактора.
- `examples/imported/simple_series_project.yaml` - минимальный YAML импорт.
- `examples/imported/sne_emrtu_project.json` - расширенный JSON импорт с компонентами, схемами, ожидаемыми результатами и предупреждениями.

## Внутренний JSON схемы редактора

Файл является объектом с полями:

- `name` - название схемы.
- `blocks` - список блоков.
- `connections` - список связей.
- `metadata` - необязательный объект с дополнительными данными.
- `version` - необязательная версия при сохранении из приложения.

Блок в `blocks`:

- `block_id` - обязательный уникальный идентификатор.
- `name` - обязательное отображаемое имя.
- `kind` - обязательный тип блока: `in`, `out`, `right`, `up`, `up_right`, `down_right`, `junction`.
- `x`, `y` - координаты на сцене.
- `params` - параметры расчета.
- `is_subscheme` - признак вложенной схемы.
- `nested_scheme` - вложенная схема в том же формате или `null`.

Связь в `connections`:

- `connection_id` - идентификатор связи, может быть пустым.
- `source_id` - обязательный `block_id` источника.
- `source_port` - порт источника, например `out`, `right`, `down_spec`.
- `target_id` - обязательный `block_id` приемника.
- `target_port` - порт приемника, например `in`, `left`, `up_spec`.

Типичные параметры в `params`:

- `lambda` - интенсивность отказов блока.
- `Tv` - среднее время восстановления.
- `P` - готовое значение вероятности безотказной работы, если оно задано таблично.
- `Kg` - готовый коэффициент готовности блока.
- `t` - расчетный горизонт.
- `block_role` - роль блока: `ordinary`, `reserve`, `k_of_n`, `subscheme`, `passive`.
- `reserve_count` - число резервных элементов для схемы `1 + reserve_count`.
- `k_required`, `n_total`, `reserve_type` - параметры группы k из n.
- `probability_by_time` - таблица вероятностей по времени, где ключи времени хранятся строками.

## Нормализованный импорт JSON/YAML

Импортированный проект является объектом с обязательными полями:

- `schema_version` - версия формата, например `"1.0"`.
- `project_name` - название проекта.
- `schemes` - одна или несколько схем.

Необязательные поля верхнего уровня:

- `source` - сведения об источнике.
- `requirements` - пороговые требования.
- `calculation_conditions` - условия расчета, например `t_values_h`.
- `components` - справочник компонентов.
- `formulas` - исходные или справочные формулы.
- `expected_results` - ожидаемые значения для проверки.
- `conclusions` - выводы.
- `warnings` - предупреждения импорта.
- `manual_review_required` - пункты, требующие ручной проверки.

`components` и `schemes` могут быть списками объектов с `id` или словарями, где ключ является id.

Компонент обычно содержит:

- `id` - идентификатор.
- `name` - имя.
- `quantity` - количество.
- `parameters` - параметры расчета.

Параметры компонента, которые реально распознает импортер:

- `lambda_work` -> преобразуется в `lambda`.
- `recovery_time` -> преобразуется в `Tv`.
- `availability` -> преобразуется в `Kg`.
- `reserve_count`.
- `k_required`.
- `n_total`.
- `block_role`.
- `reserve_type`.
- `calculation_note`.
- `probability_by_time`.

Схема в `schemes` содержит:

- `id` - идентификатор схемы.
- `name` - название.
- `type` - тип верхнего узла, обычно `series`.
- `root` - дерево структуры.
- `children` - может использоваться вместо `root.children`.
- `source_reference`, `confidence`, `notes` - необязательные метаданные.

Узлы дерева `root`:

- `type: series` - последовательное соединение `children`.
- `type: parallel` - параллельные ветви `children`.
- `type: reserve_group` - группа резерва с `k_required` и `n_total`.
- `type: nested_scheme` - ссылка на другую схему через `scheme_id`.
- строка вместо объекта - ссылка на компонент из `components`.
- обычный объект элемента - блок с `id`, `name`, `parameters`.

Дополнительно для узлов:

- `repeat` и `child` повторяют дочерний узел несколько раз.
- `quantity` вместе с `connection: series` разворачивает элемент в последовательность.

## Запуск на примере

В приложении откройте графический редактор и используйте:

- `Загрузить схему` для файлов внутренней схемы из `examples/*.json`.
- `Импорт расчета JSON/YAML` для файлов из `examples/imported/*.json` или `examples/imported/*.yaml`.
- `Демо-схема` или пункт меню `Демо` для встроенного сценария.

## Как добавить свой пример

Для простой схемы редактора удобнее сохранить ее из приложения через `Сохранить схему` и затем отредактировать JSON вручную при необходимости.

Для инженерного импорта создайте файл по образцу `examples/imported/simple_series_project.yaml`: задайте `schema_version`, `project_name`, `components`, `schemes` и при необходимости `expected_results`.

## Типичные ошибки

- Нет `schema_version` в импортированном проекте.
- Нет ни одной схемы в `schemes`.
- Связь во внутренней схеме ссылается на несуществующий `block_id`.
- В схеме больше одного входа `in` или больше одного выхода `out`.
- `kind` блока не входит в поддерживаемый набор.
- Цикл во вложенных импортированных схемах.
- `repeat` меньше 1.
- YAML импорт используется без установленного `PyYAML`.
- Значения параметров записаны сложными объектами там, где импортер ожидает число, строку, bool или `null`.
